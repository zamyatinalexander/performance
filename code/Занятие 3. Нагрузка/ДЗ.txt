ДЗ-8: Откалибровать сценарий
1. Возьмите какой-нибудь тест, содержащий две группы потоков, в одной из которых создаются некоторые объекты, а в другой эти объекты удаляются.

Можно использовать ранее разработанный тест для создания и удаления контактов в учебном приложении "адресная книга".

Можно разработать тест для своего собственного приложения -- найдите в нём такие "парные" операции, которые что-то создают и удаляют и реализуйте сценарии для создания и удаления в двух группах потоков.

2. Добавьте в сценарии задержки:
-- перед каждым запросом, который является результатом прохода пользователя по ссылке, вставьте задержку продолжительностью примерно 1 секунда,
-- перед каждым запросом, который является результатом отправки формы, вставьте задержку продолжительностью 3-5 секунд.

3. Откалибруйте сценарий, уменьшая или увеличивая количество потоков и используя дополнительные задержки, чтобы при достаточно длительном выполнении (5-10-15 минут) количество транзакций создания было примерно равно количеству транзакций удаления.

Для контроля количества выполняемых транзакций используйте наблюдатель (listener) Summary Report.

В качестве ответа на задание приложите архив, содержащий jmx-файл со сценариями, необходимые для его работы файлы с тестовыми данными, а также скриншот компонента Summary Report, показывающий количество транзакций разного типа.



ДЗ-9: Поискать точки насыщения
При помощи ранее созданного откалиброванного теста попробуйте поискать точки насыщения, используя профиль нагрузки с равномерно увеличивающимся количеством виртуальных пользователей (потоков).

Внимание: перед тем, как выполнять такой тест на собственном приложении, обязательно согласуйте свои действия с коллегами и руководством!

Для того, чтобы заметить точку насыщения, можно использовать сочетание компонента Summary Report (там будет хорошо видно, если начали появляться отказы или если какой-то показатель очень резко начал возрастать) и какого-нибудь компонента, который рисует графики, либо Graph Results (встроен в JMeter), либо Response Times Over Time (устанавливается через менеджер плагинов).

В качестве ответа на задание приложите отчёт в произвольной форме, в котором описано, какие тесты выполнены и какие результаты получены.



ДЗ-10: Выполните "ручное" тестирование производительности
Для своего собственного приложения или для какого-нибудь другого сайта выполните тестирование производительности загрузки нескольких страниц (выберите 2-3 страницы, которые достаточно сильно отличаются друг от друга) при помощи встроенных в браузер средств, а также при помощи дополнения YSlow для браузера Chrome.

Проанализируйте результаты, составьте отчёт в произвольной форме, опишите в нём наиболее серьёзные проблемы, которые Вы заметили, и предложите план улучшения производительности загрузки страниц.
